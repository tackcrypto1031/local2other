新增遠端主機

🔐 SSH 設定指南

在遠端電腦安裝 OpenSSH Server

以管理員身份開啟 PowerShell，執行以下指令：

1. 安裝 OpenSSH Server
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
2. 啟動 SSH 服務
Start-Service sshd
3. 設定開機自動啟動
Set-Service -Name sshd -StartupType 'Automatic'
4. 確認防火牆規則（通常安裝時會自動設定）
Get-NetFirewallRule -Name *ssh*
💡 提示：若防火牆規則不存在，執行：
New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

在本機生成 SSH 金鑰

設定金鑰後，可實現免密碼自動傳輸。

檢查金鑰狀態...
手動生成指令（在 PowerShell 執行）
ssh-keygen -t rsa -b 4096

將公鑰部署到遠端電腦

⚠️ 重要：管理員帳戶

如果遠端使用者是管理員帳戶，公鑰必須放在：

C:\ProgramData\ssh\administrators_authorized_keys

（不是 C:\Users\使用者\.ssh\authorized_keys）

步驟一：複製公鑰
載入公鑰...
步驟二：在遠端電腦執行以下指令

以管理員身份開啟 PowerShell，依序執行：

1. 建立 ssh 目錄（如果不存在）
New-Item -Path 'C:\ProgramData\ssh' -ItemType Directory -Force
2. 建立空白的 authorized_keys 檔案
New-Item -Path 'C:\ProgramData\ssh\administrators_authorized_keys' -ItemType File -Force
3. 用記事本開啟並貼上公鑰
notepad C:\ProgramData\ssh\administrators_authorized_keys
4. 設定正確的檔案權限（必要！）
icacls 'C:\ProgramData\ssh\administrators_authorized_keys' /inheritance:r /grant 'SYSTEM:F' /grant 'Administrators:F'
💡 一般使用者帳戶：

如果遠端使用者不是管理員，則公鑰放在：

C:\Users\使用者名稱\.ssh\authorized_keys

SSH 連線疑難排解

⚠️ 重要：必須使用本機帳戶

當遠端電腦加入網域時，必須使用本機帳戶，不能使用網域帳戶。

帳戶類型 範例 支援
本機帳戶 auer, Administrator ✅ 可用
網域帳戶 DOMAIN\user, user@domain.com ❌ 不支援

原因：Windows OpenSSH 服務無法為網域用戶創建認證 Token（S4U 認證限制）。

如何查找本機帳戶
在遠端電腦執行
net user
進階除錯步驟

如果連線仍然失敗，可以啟動 sshd 除錯模式查看詳細錯誤：

1. 停止 sshd 服務
Stop-Service sshd
2. 以除錯模式啟動
& 'C:\Windows\System32\OpenSSH\sshd.exe' -d
3. 在另一個終端測試連線
ssh user@192.168.x.x 'echo SUCCESS'
💡 提示：

查看 sshd -d 視窗的輸出，可以看到詳細的認證過程和錯誤訊息。
